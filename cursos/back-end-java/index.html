2. Requisitos de acesso

  • Idade mínima: 16 anos.
  • Conhecimento específico: HTML, Bootstrap e Lógica de Programação.
  • Documentos: identidade, CPF e autodeclaração de conhecimento em HTML, Bootstrap e Lógica de Programação.
  • Menores de 18 anos acompanhados de responsável legal.

3. Justificativa

Tecnologia foi um dos grandes destaques de contratação no triênio 2020–2022, acelerada pela digitalização das operações na pandemia. Vagas de desenvolvimento Java estão entre as mais difíceis de preencher (cerca de 46% ficam abertas por mais de 2 meses, segundo o Indeed). O curso visa o aperfeiçoamento em uma das linguagens mais utilizadas mundialmente, abrindo oportunidades concretas de emprego.

4. Objetivo

Que os estudantes desenvolvam aplicações back-end para web utilizando a plataforma Java — configuração do ambiente de trabalho, aplicação de banco de dados em sistemas back-end e empacotamento de um sistema. Destina-se a quem deseja atuar de forma autônoma, temporária ou efetiva em organizações públicas e privadas.

5. Organização curricular

Unidade curricular única: UC 5 — Desenvolver aplicações back-end para web — 84h.

Indicadores da UC: (1) configura o ambiente de desenvolvimento; (2) cria estruturas de código para back-end; (3) implementa session e cookies; (4) integra banco de dados; (5) gera o pacote da aplicação web.

Elemento Conteúdo
Conhecimentos Linguagem de programação back-end e orientação a objetos (sintaxe, definições, funcionalidades); tratamento de dados em Java (variáveis, constantes, tipos, operadores, expressões); estruturas de dados (vetores e matrizes); arquitetura web back-end ↔︎ front-end — Servlets, JSP, HTML/XHTML, diretivas, Scriptlets, JSTL, Expression Language, Maven, Tomcat; ambiente de desenvolvimento; biblioteca da linguagem; sessões (cookies/sessions); segurança da informação; banco de dados (SQL Server); manipulação de dados em Java (querys, funções de BD e ORM); empacotamento das fontes e deploy.
Habilidades Registrar e organizar informações e códigos; testar a aplicação web; aplicar comandos SQL; manipular dados via ORM; compilar código-fonte.
Atitudes/Valores Sigilo no tratamento de dados; iniciativa na proposição de soluções; proatividade no desenvolvimento das atividades.

6. Orientações metodológicas

Aprendizagem com autonomia e desenvolvimento de competências, com prática contextualizada. Recomenda-se integrar os conceitos de programação à orientação a objetos desde o início, trabalhar elementos comuns a sistemas web administrativos/comerciais, priorizar a prática no uso de banco de dados e empregar frameworks que facilitem a construção do back-end, especialmente na integração com o banco.

7. Avaliação

Avaliação diagnóstica, formativa e somativa. Menções por indicador (A / PA / NA) e por UC (Desenvolvida — D / Não desenvolvida — ND). Aprovação exige D em todas as UCs e frequência mínima de 75% (menções finais AP / RP). Recuperação imediata à constatação de dificuldades.

8–12. Recursos, perfil e certificação

1 computador por aluno (i5 10ª geração+, Windows 10 Pro, 8 GB, SSD 256 GB) com internet, projetor e softwares atualizados: VS Code, Spring Tool Suite (STS), Star UML, Draw.io, MS Visio, SQL Server + Management Studio e Git. Sem material didático fornecido/adquirido. Docente com experiência em lógica de programação, banco de dados e programação Java. Certificado de Aperfeiçoamento em Desenvolvedor Back-End Java, validade nacional.

Fonte oficial: PTD — Desenvolvedor Back-End Java (.docx)

A UC é desenvolvida ao longo de 7 situações de aprendizagem (84h), ancoradas em casos de empresas fictícias que evoluem do “Olá mundo” até um sistema web integrado a banco de dados. IDEs sugeridas: NetBeans/Eclipse (o PC oficial referencia Spring Tool Suite).

Situações de aprendizagem (84h)

SA CH Caso / foco
1 8h Software’s Quality — instalação do ambiente (JDK, IDE), lógica, saídas (System.out), variáveis, operadores.
2 4h Software’s Quality — armazenamento dinâmico de dados e estruturas de repetição (rotinas).
3 16h Escola do Aprender — levantamento de requisitos, classes e objetos preparando a futura conexão com interfaces.
4 4h Escola do Aprender — interfaces gráficas e layouts conforme as regras de negócio.
5 20h Books Store — e-commerce / páginas web em Java, requisitos e preparação para conexão ao banco.
6 20h EquipaBem — sistema online multifilial, diagrama de classes, cadastros e controle de acesso.
7 12h Rede de academias — sistema online com integração a banco de dados e interação entre filiais.

Notas para o docente

Reforçar o estudo de orientação a objetos e suas vantagens; usar a SA 1 também como avaliação diagnóstica do nível da turma; ressaltar as Marcas Formativas Senac (domínio técnico-científico, visão crítica, atitude empreendedora/sustentável/colaborativa, foco em resultados) com o aluno como protagonista. Recurso didático recorrente: laboratório com internet e IDE configurada com o JDK.
