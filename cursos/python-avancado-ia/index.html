  • Eixo Tecnológico: Informação e Comunicação · Segmento: Tecnologia

2. Requisitos de acesso

  • Idade mínima: 16 anos · Escolaridade: Ensino Médio completo.
  • Conhecimentos específicos: Python; desenvolvimento web com Django; manipulação básica de banco de dados; controle de versão com Git.
  • Documentos: identidade, CPF, comprovante de escolaridade, comprovante ou autodeclaração de conhecimentos específicos.

3. Justificativa

O mercado de tecnologia demanda profissionais que dominem o ecossistema de desenvolvimento moderno — criação de APIs, conteinerização e aplicação prática de IA. O curso é o próximo passo natural após Programação em Python, transformando o programador em um profissional preparado para criar aplicações complexas e com recursos de IA.

4. Objetivo

Possibilitar ao aluno a criação de APIs profissionais integrando modelos de IA (LLMs) para funcionalidades inovadoras. Destina-se a egressos de Programação em Python do Senac RJ ou a quem tenha conhecimento equivalente.

5. Organização curricular

Curso sem comprometimento integral com competências: a Unidade Curricular é o próprio curso — Programação em Python avançado com IA (100h).

Indicador: implementa soluções completas com conteinerização, implantação contínua e integração de LLMs, seguindo boas práticas de IA.

Elemento Conteúdo
Conhecimentos POO avançado (herança múltipla, Mixins); decoradores, geradores e iteradores; Docker (Dockerfile, isolamento de dependências) e docker-compose; Django REST Framework (endpoints CRUD com Views/ViewSets, serialização, autenticação e permissões); FastAPI; sistemas de cache; fila de mensageria (RabbitMQ ou Kafka); GitHub Actions (CI/CD); documentação de APIs (Postman/Insomnia + Swagger); Streamlit + SQL com LLM para gerar consultas.
Habilidades Elabora arquitetura MVC; aplica regras de negócio na camada de serviço; integração/mapeamento objeto-relacional; testes automatizados; constrói contêineres e integração/implantação contínua com GitHub Actions; integra APIs de LLM para interpretar perguntas em linguagem natural.

6. Orientações metodológicas

Aprendizagem com autonomia e prática pedagógica contextualizada, orientada pelas Marcas Formativas Senac — com destaque para a Autonomia digital (uso crítico de IA). Apoio nos recursos do Espaço Docente (Cachola, Cubus, Anuário de Tecnologias Educacionais). Percurso específico: POO avançado → conteinerização → DRF/FastAPI → cache e mensageria → documentação → Streamlit + LLM.

7. Avaliação

Menções de processo Atendido (A) / Parcialmente Atendido (PA) / Não Atendido (NA); ao final, A / NA. Resultado do curso: Concluiu (C) com indicadores atendidos e frequência ≥ 75%; caso contrário, Não Concluiu (NC). Recuperação oportunizada ao longo do processo.

8–11. Recursos, perfil e certificação

1 computador por aluno (i5 10ª geração+, Windows 11, 8 GB RAM, SSD 256 GB, internet ≥ 100 Mbps) com VS Code, Django, FastAPI, Docker, Postman, PyCharm, MySQL, GitHub Actions e GitHub Copilot. Sem material didático fornecido/adquirido. Docente com experiência em back-end Python (Django/DRF e FastAPI, APIs RESTful), bancos relacionais (SQLite/MySQL), cache (Redis) e mensageria (RabbitMQ/Kafka). Certificado de Aperfeiçoamento em Programação em Python avançado com IA, validade nacional.

Fonte oficial: PTD — Programação Avançada em Python com IA (.docx)

Situação de aprendizagem: a empresa fictícia SynapseTech abre vaga de Programador de IA com foco em back-end; ao longo do curso os alunos, em times, constroem a solução em etapas — como num processo seletivo por desafios. Indicador da UC: implementa soluções completas com conteinerização, automação CI/CD e integração de LLMs, seguindo padrões DevOps e boas práticas de IA.

Procedimentos/instrumentos de avaliação: prática que simula um case real de mercado, em grupos, com apresentação dos resultados e feedbacks. Instrumentos: documentação técnica, código-fonte versionado, planejamento em grupo e demonstração da solução. A atividade avaliativa formal ocorre nas aulas 16/17/18 (verificação dos conhecimentos da UC).

Planejamento das aulas

O PTD detalha o percurso aula a aula (cada aula com sugestão de corte de 2h, para ajuste de calendário). A IA generativa entra como copiloto/parecerista em todas as etapas — gerar rascunhos, revisar criticamente a saída e registrar os prompts (PROMPTS.md).

Bloco Foco Onde a IA entra
1–2 Setup (Python/Git/Docker/IDE), GitHub Projects; arquitetura em camadas (API/Service/Repository), DTOs/Serializers IA gera estrutura do projeto a partir do diagrama; revisão crítica da saída.
3–5 Docker (Dockerfile multistage, docker-compose); DRF (model→serializer→viewset→router); FastAPI + Pydantic IA como parecerista de código e templates de prompt para o time.
6–8 Modelagem relacional e migrações; autenticação JWT/roles/throttling; cache-aside (Redis), TTL e invalidação Boas práticas e checklist IA-safe.
9–11 Mensageria — RabbitMQ e Kafka (idempotência, DLQ, reentrega); desafio em grupo (mini-backend de pedidos) Comparação de brokers e trade-offs.
12–13 Testes (pytest, fixtures, cobertura); documentação OpenAPI/Swagger, ReDoc e coleção Postman IA gera testes e exemplos; revisão cruzada doc ↔︎ código.
14–15 Integração de LLMs (timeouts, retry, circuit breaker, telemetria, LGPD/redaction); engenharia de prompts (zero-shot, few-shot, chain-of-thought) Núcleo de IA do curso: llm_service e endpoint /assist.
16–18 Avaliação em grupo — solução completa (banco, contêiner, CRUD, cache, mensageria, Swagger, README) Aplicação integrada dos conteúdos.
19–24 Streamlit + SQL seguro (consultas parametrizadas, anti-SQL injection); dashboards; CI/CD com GitHub Actions (build, deploy DockerHub); text-to-SQL com LLM; introdução a RAG (embeddings, índice, citações) IA traduz pergunta em SQL; pipeline RAG sobre os próprios docs da API.

Orientações gerais

Validar previamente permissões e dependências nas máquinas; documentar as rotas em Postman/Insomnia; salvar todo o conteúdo em repositórios; tratar momentos de construção conjunta como prática orientada. O sistema construído no desafio da aula 11 é reaproveitado nas aulas 20 e seguintes.
