1. Identificação

  • Eixo: Informação e Comunicação · Segmento: Tecnologia da Informação
  • Tipo: Aperfeiçoamento · Carga horária: 360 horas · Código DN: 986.82 · Código Senac RJ: 13323 (cadastrado em 09.10.2023)

2. Requisitos de acesso

  • Idade mínima: 16 anos · Escolaridade: Ensino Médio completo (ou cursando o último ano).
  • Noções básicas de informática (sistemas operacionais e navegadores de internet).
  • Documentos: identidade, CPF, comprovante de escolaridade e autodeclaração de noções básicas de informática.

3. Justificativa

Curso voltado à forte demanda do mercado por desenvolvedores web/full stack. Foi desenvolvido pelo Senac RJ em colaboração com a Assespro-RJ e o Serratec (Parque Tecnológico da Região Serrana), visando às habilidades necessárias para a atuação profissional em Tecnologia da Informação.

4. Objetivo

Subsidiar os participantes na publicação de aplicações web, estruturando o front-end e desenvolvendo o back-end com ênfase na plataforma Java. Destinado a quem deseja atuar como autônomo, temporário ou efetivo, em organizações públicas e privadas.

5. Organização curricular

Sete unidades curriculares, totalizando 360h (a UC7 — Projeto Integrador — atravessa as demais):

Unidade Curricular CH
UC1 — Desenvolver soluções utilizando algoritmos, lógica e linguagem de programação (JavaScript/TypeScript) 36h
UC2 — Desenvolver Soft Skills para o Mercado de Trabalho 36h
UC3 — Elaborar projetos de aplicações web (briefing, metodologias ágeis, UML, arquitetura da informação) 36h
UC4 — Estruturar aplicações front-end para web (HTML semântico, CSS, JavaScript/AJAX, Bootstrap, UX/responsividade) 72h
UC5 — Desenvolver aplicações back-end para web com Java (OO, Maven/Tomcat, sessões/cookies, SQL Server, ORM, deploy) 84h
UC6 — Desenvolver aplicações híbridas para mobile (Flutter/Dart, widgets, multiplataforma) 60h
UC7 — Projeto Integrador (ação-reflexão-ação) 36h

Cada UC detalha Conhecimentos / Habilidades / Atitudes-Valores e indicadores próprios no PC. Em síntese: a UC1 firma lógica e algoritmos; a UC3 cobre planejamento e documentação do projeto; UC4 e UC5 entregam o eixo front/back; a UC6 estende para mobile híbrido; e a UC7 articula tudo em temas geradores reais (ex.: aplicação web acessível multiplataforma; site de portfólio de produtos/serviços).

6. Orientações metodológicas

Aprendizagem por desenvolvimento de competências e situações-problema contextualizadas, ancorada nas Marcas Formativas Senac (domínio técnico-científico, visão crítica, atitude empreendedora, sustentável e colaborativa). O Projeto Integrador é executado em interação entre os docentes, preferencialmente sobre demandas reais de empresas da região, com ferramentas gratuitas de gestão/colaboração.

7. Avaliação

Avaliação diagnóstica, formativa e somativa. Menções por indicador (A / PA / NA), por UC (Desenvolvida — D / Não desenvolvida — ND) e de curso (Aprovado — AP / Reprovado — RP). Aprovação exige D em todas as UCs e frequência mínima de 75%. Recuperação é imediata à constatação das dificuldades.

8–11. Recursos, perfil e certificação

1 computador por aluno (i5 10ª geração+, Windows 10 Pro, 8GB, SSD 256GB) e laboratório com internet e projetor. Softwares: VS Code, Spring Tool Suite, Node.js, Hyper-V, StarUML, Draw.io, VISIO, Flutter, Dart, Office, SQL Server + Management Studio, Ionic, Git, JDK 11, Android Studio. Docente com experiência em lógica de programação, back-end, front-end e mobile, e formação na área de TI. Aproveitamento de estudos anteriores via protocolo de avaliação de competências. Certificado de Aperfeiçoamento em Programador Full Stack, com validade nacional.

Fonte oficial (um PTD por UC):

  • PTD — UC1 (.docx) · UC2 · UC3 · UC4 · UC5 · UC6 · UC7 — Projeto Integrador

Cada PTD organiza a UC em situações de aprendizagem mapeadas aos indicadores, com cronograma de aulas (carga horária da UC), detalhamento de atividades de aprendizagem, recursos didáticos e a contribuição da UC para o Projeto Integrador.

Estrutura comum dos PTDs

  • Cabeçalho: curso, código SGA, indicadores e tema gerador do PI; contribuição da UC para o PI.
  • Situações de aprendizagem: descrição contextualizada do problema/desafio, elementos de competência (Conhecimentos / Habilidades / Atitudes) e desdobramento aula a aula (ex.: UC1 abre com avaliação diagnóstica, debate de mercado e o jogo “Torre de Hanói” para lógica).
  • Avaliação: processual e contínua (diagnóstica → formativa por indicadores → somativa), com instrumentos como a Ficha de Avaliação Individual.

Nota: os PTDs trazem cabeçalho de versão anterior do curso (“Formação - Programação Java”, SGA 12841), mas o conteúdo das UCs corresponde ao PC vigente (SGA 13323).
